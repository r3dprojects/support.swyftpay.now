# Security Policy

## Reporting a vulnerability

**Do not open a public GitHub issue for security vulnerabilities.**

Use one of the following channels:

- **GitHub Private Vulnerability Reporting** (preferred): [Report a vulnerability](https://github.com/r3dprojects/support.swyftpay.now/security/advisories/new)
- **Email**: [info@swyftec.com](mailto:info@swyftec.com)

Please include as much detail as you can: steps to reproduce, impact assessment, and any proof-of-concept. We will acknowledge receipt within 2 business days and keep you informed of progress.

We do not run a public bug bounty programme at this time.

## Scope

Reports are accepted for the production swyftpay.now service and associated infrastructure. This repository contains no executable code — it is an issue tracker only.

## Disclosure policy

We ask that you give us reasonable time to investigate and remediate before any public disclosure. We aim to resolve critical issues within 30 days of a confirmed report.
